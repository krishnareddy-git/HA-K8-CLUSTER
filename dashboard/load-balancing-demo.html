How It Works

Kubernetes Service Load Balancing

When you create a Kubernetes Service, it acts as a stable endpoint with its own ClusterIP. The Service automatically distributes incoming traffic across all healthy pods that match its selector.

Round Robin Algorithm

The default algorithm. Each request is sent to the next pod in sequence. This ensures even distribution when all pods have similar capacity.

Health Checks & Failover

Kubernetes continuously monitors pod health. If a pod fails, traffic is automatically redirected to healthy pods only. Try disabling a pod to see this in action!

Types of Load Balancing

  • Service LB: Distributes traffic to pods within the cluster
  • HAProxy: External load balancer for API server HA
  • Ingress: HTTP/HTTPS routing and load balancing
